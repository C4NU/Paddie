# Paddie 복구 백로그

기준 버전: `v3.4.1` (`b906a85`)

## 목표

- 안정 버전으로 확인된 `v3.4.1`을 기준으로 데스크톱 앱을 복구한다.
- `v3.4.1` 이후 수정은 작고 검토 가능한 단위로만 다시 적용한다.
- UI 변경은 인코딩, 리소스 경로, 빌드, 변환 로직 수정과 분리하되 PySide6 기준으로 안정화한다.

## v3.4.1 이후 주요 회귀 원인

- Windows에서 UTF-8 메타데이터 파일을 기본 인코딩으로 읽으면서 `cp949` 오류가 발생했다.
- `resources/` 구조가 이후 `resources/data`, `resources/i18n`, `resources/icons`로 재배치되었다.
- 리소스 이동 뒤 PyInstaller spec의 참조 경로가 일부 어긋났다.
- i18n 리소스 이동 뒤 초기 번역 로드 경로와 런타임 언어 변경 경로가 달라졌다.
- 변환 작업을 worker로 옮기면서 리사이즈 축과 설정값 연결이 틀어질 위험이 생겼다.
- 자동 업데이트와 분리 프리뷰 기능이 앱 시작 및 런타임 경로를 크게 늘렸다.
- Flet 실험이 기존 Qt 의존성과 섞였다.

## 우선 다시 적용할 후보 수정

1. `program_data.json`을 다시 도입하는 경우 `encoding="utf-8"`을 명시한다.
2. `model_map.csv`를 읽을 때 `encoding="utf-8"`을 명시한다.
3. PySide6 단일 의존성으로 전환하고 `.ui` 로딩을 검증한다.
4. 복구 브랜치에서 버전 메타데이터를 어떤 파일로 관리할지 먼저 정한다.
5. 현재 `resources/data`, `resources/fonts`, `resources/icons`, `resources/ui` 구조를 기준으로 최소 빌드 스모크 체크를 추가한다.
6. 추가 리소스 구조 변경은 별도 커밋으로만 검토한다.

## 처리한 기준 버전 경고

- `src/exif_module.py`에서 정수 alignment 값을 `is`로 비교한다.
- `src/resize_module.py`에서 정수 axis option 값을 `is`로 비교한다.
- 복구 브랜치에서 `==` 비교로 정리했다.

## 보류

- Flet UI.
- wxPython 마이그레이션.
- 자동 업데이트 재작성.
- 전체 i18n 구조 개편.

## main에서 다시 가져올 기능 후보

### 우선 적용 후보

1. 정보 창 표시값 보정
   - 범위: 프로그램명, 버전, 연락처 라벨 보정.
   - 상태: 적용 완료.
   - 상태: 정보 창 레이아웃을 아이콘/설명/버전/메타데이터 중심의 코드 기반 디자인으로 교체했다.
   - 상태: 라이선스 파일이 아직 없어 정보 창에는 `별도 명시 없음`으로 표시한다.

2. Windows 인코딩 안정화
   - 범위: JSON/CSV 파일 읽기와 쓰기에 UTF-8 명시.
   - 상태: 적용 완료.

3. PyInstaller 최소 빌드 설정
   - 범위: 재배치된 `resources/data`, `resources/fonts`, `resources/icons`, `resources/ui`, `resources/i18n` 구조를 기준으로 빌드 명령을 정리한다.
   - 상태: `scripts/build_release.py`와 `scripts/package_release.py`로 공통 빌드/패키징 경로를 만들었다.
   - 상태: `.venv/bin/python scripts/build_release.py` 기준 macOS 빌드 통과.

4. PySide6 단일화
   - 범위: PyQt6 의존성을 제거하고 기존 `.ui` 파일을 PySide6 `QUiLoader`로 읽는다.
   - 상태: 코드 전환 적용, PyInstaller 빌드 통과.
   - 주의: PySide6는 일부 Qt static 메서드에서 keyword 인자를 받지 않으므로 positional 호출을 유지한다.

5. 로컬라이징 최소 복원
   - 범위: Qt `QTranslator`와 `.qm` 파일 기반으로 앱 시작 및 설정 변경 시 언어를 적용한다.
   - 상태: `resources/i18n` 복원, 앱 시작 시 번역 설치, 설정 창을 통한 실시간 번역 교체를 적용했다.
   - 주의: 영어는 원문 UI 언어로 처리하고, 미완성 번역이 빈 문자열을 반환하면 원문으로 fallback한다.
   - 주의: 자체 JSON translator는 도입하지 않는다.

6. GitHub Actions 빌드 스모크 체크
   - 범위: `.venv` 또는 CI 가상환경 의존성 설치와 PyInstaller 빌드까지만.
   - 상태: macOS Intel, macOS Apple Silicon, Windows x64, Linux x64 빌드 matrix를 추가했다.
   - 상태: 태그 `v*` 빌드에서는 산출물을 첨부한 draft release를 생성한다.
   - 주의: 원격 GitHub Actions 실행 결과는 아직 확인하지 않았다.

### 검토 후 적용 후보

1. 설정 창
   - 장점: 언어 같은 사용자 설정 진입점을 만들 수 있다.
   - 위험: i18n과 실시간 설정 반영이 엮이면 범위가 커진다.
   - 상태: 언어 선택만 먼저 추가했고, 선택 후 앱 재실행 없이 현재 로드된 Qt UI 텍스트를 다시 번역한다.
   - 상태: macOS 앱 메뉴로 이동해야 하는 Preferences/Information/Exit 액션은 번역 문자열 대신 Qt menu role로 고정했다.

2. 변환 worker 도입
   - 장점: 대량 변환 중 UI 멈춤을 줄인다.
   - 위험: `main` 구현에는 리사이즈 축 연결 버그가 있었다.
   - 기준: 기존 동기 변환과 결과가 같은지 먼저 작은 샘플로 검증한다.

3. 프로그램 메타데이터 파일
   - 장점: 정보 창 버전/연락처를 코드 밖에서 관리할 수 있다.
   - 위험: `program_data.json` 도입 시 경로와 인코딩 문제가 다시 생길 수 있다.
   - 기준: `resources/data/program_data.json` 단일 위치로 시작하고 UTF-8을 명시한다.

4. 업데이트 확인
   - 장점: GitHub Releases의 최신 버전을 안내할 수 있다.
   - 위험: 자동 설치까지 한 번에 붙이면 앱 교체와 롤백 문제가 커진다.
   - 상태: 최신 릴리스 확인과 브라우저 열기까지만 구현했다.

5. 분리 프리뷰 창
   - 장점: 큰 프리뷰 때문에 메인 창이 과도하게 커지는 문제를 줄인다.
   - 상태: 기존 EXIF 미리보기 생성 경로를 재사용하고 항상 별도 창으로 표시하도록 적용.
   - 남은 일: 새 빌드 산출물에서 실제 이미지 샘플로 별도 창 표시를 확인한다.

### 보류 후보

1. 자동 업데이트 설치
   - 이유: 네트워크, 릴리스 자산 구조, 앱 교체 로직, 롤백 처리가 한 번에 들어간다.
   - 상태: 자동 설치는 아직 구현하지 않았다.

2. 추가 리소스 구조 확장
   - 이유: 현재 재배치 범위를 넘어서는 `i18n`과 빌드 자산 확장은 경로 참조 누락 위험이 크다.

3. 전체 i18n 구조
   - 이유: 최소 Qt 번역 복원과 런타임 언어 변경은 적용했지만, 자체 JSON 번역 구조까지 넓히는 것은 별도로 검토한다.

4. 실시간 프리뷰 자동 동기화
   - 이유: 설정 변경 감지, 임시 출력 파일, 창 상태 관리가 함께 필요하다.
