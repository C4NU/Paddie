# Paddie 복구 백로그

기준 버전: `v3.4.1` (`b906a85`)

## 목표

- 안정 버전 `v3.4.1`을 기준으로 데스크톱 앱을 복구한다.
- 이후 수정은 작고 검토 가능한 단위로 다시 적용한다.
- UI 변경은 인코딩, 리소스, 빌드, 변환 수정과 분리한다.
- PySide6 기준으로 안정화한다.

## v3.4.1 이후 주요 회귀 원인

- Windows에서 UTF-8 메타데이터를 기본 인코딩으로 읽었다.
- 이로 인해 `cp949` 오류가 발생했다.
- `resources/` 구조가 `data`, `i18n`, `icons`로 재배치되었다.
- 리소스 이동 뒤 PyInstaller spec의 참조 경로가 어긋났다.
- i18n 리소스 이동 뒤 초기 번역 로드 경로가 달라졌다.
- 런타임 언어 변경 경로도 초기 로드 경로와 달라졌다.
- 변환 worker 도입으로 리사이즈 축 연결 위험이 생겼다.
- 업데이트 확인과 분리 프리뷰가 앱 시작 경로를 늘렸다.
- Flet 실험이 기존 Qt 의존성과 섞였다.

## 우선 다시 적용할 후보 수정

1. `program_data.json` 재도입 시 `encoding="utf-8"`을 명시한다.
1. `model_map.csv`를 읽을 때 `encoding="utf-8"`을 명시한다.
1. PySide6 단일 의존성으로 전환하고 `.ui` 로딩을 검증한다.
1. 버전 메타데이터를 어떤 파일로 관리할지 먼저 정한다.
1. 현재 리소스 구조 기준으로 빌드 스모크 체크를 추가한다.
1. 추가 리소스 구조 변경은 별도 커밋으로만 검토한다.

## 처리한 기준 버전 경고

- `src/exif_module.py`에서 정수 alignment 값을 `is`로 비교했다.
- `src/resize_module.py`에서 정수 axis option 값을 `is`로 비교했다.
- 복구 브랜치에서 두 비교를 `==`로 정리했다.

## 보류

- Flet UI.
- wxPython 마이그레이션.
- 자동 업데이트 재작성.
- 전체 i18n 구조 개편.

## main에서 다시 가져올 기능 후보

### 우선 적용 후보

1. 정보 창 표시값 보정

   - 범위: 프로그램명, 버전, 연락처 라벨 보정.
   - 상태: 적용 완료.
   - 상태: 정보 창을 코드 기반 디자인으로 교체했다.
   - 상태: 라이선스 파일이 없어 미지정으로 표시한다.

1. Windows 인코딩 안정화

   - 범위: JSON/CSV 파일 읽기와 쓰기에 UTF-8 명시.
   - 상태: 적용 완료.

1. PyInstaller 최소 빌드 설정

   - 범위: 현재 리소스 구조 기준으로 빌드 명령을 정리한다.
   - 상태: 공통 빌드/패키징 스크립트를 만들었다.
   - 상태: macOS 로컬 빌드가 통과했다.

1. PySide6 단일화

   - 범위: PyQt6 의존성을 제거하고 `QUiLoader`를 사용한다.
   - 상태: 코드 전환 적용, PyInstaller 빌드 통과.
   - 주의: Qt static 메서드 호출은 positional 인자를 유지한다.

1. 로컬라이징 최소 복원

   - 범위: Qt `QTranslator`와 `.qm` 파일로 언어를 적용한다.
   - 상태: 시작 시 번역 설치와 실시간 교체를 적용했다.
   - 주의: 영어는 원문 UI 언어로 처리한다.
   - 주의: 빈 번역 문자열은 원문으로 fallback한다.

1. GitHub Actions 빌드 스모크 체크

   - 범위: CI 의존성 설치와 PyInstaller 빌드까지만.
   - 상태: 4개 플랫폼 빌드 matrix를 추가했다.
   - 상태: 태그 `v*` 빌드는 draft release를 생성한다.
   - 주의: 원격 Actions 실행 결과는 아직 확인하지 않았다.

### 검토 후 적용 후보

1. 설정 창

   - 장점: 사용자 설정 진입점을 만들 수 있다.
   - 위험: i18n과 실시간 반영이 엮이면 범위가 커진다.
   - 상태: 언어 선택만 먼저 추가했다.
   - 상태: 재실행 없이 현재 Qt UI 텍스트를 다시 번역한다.
   - 상태: macOS 앱 메뉴 액션은 Qt menu role로 고정했다.

1. 변환 worker 도입

   - 장점: 대량 변환 중 UI 멈춤을 줄인다.
   - 위험: `main` 구현에는 리사이즈 축 연결 버그가 있었다.
   - 기준: 기존 동기 변환과 결과가 같은지 먼저 검증한다.

1. 프로그램 메타데이터 파일

   - 장점: 정보 창 값을 코드 밖에서 관리할 수 있다.
   - 위험: 경로와 인코딩 문제가 다시 생길 수 있다.
   - 기준: `resources/data/program_data.json` 단일 위치로 시작한다.

1. 업데이트 확인

   - 장점: GitHub Releases의 최신 버전을 안내할 수 있다.
   - 위험: 자동 설치까지 붙이면 롤백 문제가 커진다.
   - 상태: 최신 릴리스 확인과 브라우저 열기만 구현했다.

1. 분리 프리뷰 창

   - 장점: 큰 프리뷰로 메인 창이 커지는 문제를 줄인다.
   - 상태: 항상 별도 창으로 표시하도록 적용했다.
   - 남은 일: 실제 이미지로 별도 창 표시를 확인한다.

### 보류 후보

1. 자동 업데이트 설치

   - 이유: 네트워크, 앱 교체, 롤백 처리가 필요하다.
   - 상태: 자동 설치는 아직 구현하지 않았다.

1. 추가 리소스 구조 확장

   - 이유: 빌드 자산 확장은 경로 참조 누락 위험이 크다.

1. 전체 i18n 구조

   - 이유: 자체 JSON 번역 구조는 별도로 검토한다.

1. 실시간 프리뷰 자동 동기화

   - 이유: 설정 감지, 임시 출력, 창 상태 관리가 필요하다.
